Shows possible cases that can happen during reconciliation of rows for a hypotetical entity type Tourism Resort .

Instructions

  • This is an interactive version of the mockups. You can also download a pdf version (some non essential image parts are voluntarily cropped in the pdf version)
  • Hover on red circles to see a description of the relative item
  • Only Firefox browser with a recent version is supported

Assumptions

  • incomplete entities are allowed - that is, there are not enough attributes to satisfy at least an identifieng set
  • inconsistent entities are allowed - it is possible to insert in the database an entity which has one identifying set equals to an existing entity but another identifying set which is different and cannot be merged

Table description

First five columns represent attributes from the dataset. Last two columns are other attributes from the etype. First seven rows only consider possible id results without considering eventual value updates and automerges. In last row (Moena) some GUI is proposed to visualize updated values and provenance. Actual editing and merging should be done in a separate Entity alignment panel. Notice project name bar and facets on the left were removed from this pdf version for space reasons.
